Ensure technical explanations are clear, accessible, and accurate without sacrificing precision.

WHAT IT DOES:
Reviews and improves technical writing for clarity. Identifies jargon, suggests better explanations, checks readability, ensures logical flow.

CLARITY PRINCIPLES:
- Define technical terms on first use
- Use analogies for abstract concepts
- Break long sentences into shorter ones
- Use active voice over passive
- Provide examples after definitions
- Use bullet points for lists
- Add diagrams/visuals where helpful (describe them)
- Maintain consistent terminology

CHECKS:
✓ Is this understandable to the target audience?
✓ Are terms defined before use?
✓ Are sentences concise (<25 words)?
✓ Is the logic flow clear?
✓ Are examples concrete and relevant?

USE FOR: Reviewing all written content, improving explanations, making concepts accessible.
